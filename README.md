#Portfolio Sida

Detta är en portfolio sida som presenterar en profil och vilka egenskaper och skickligheter som denna profil erhåller i form av livserfarenhet, utbildning och projekt. Sidan går även igenom vilka tech stackar och program som profilen använder samt visar upp exempel på projekt personen har skapat. Webbsidan innehåller fem huvudsidor vid namnen “home”, “about”, “tech stack”, “projects” och “contact”. Syftet med webbplatsen är att fungera som en digital portfolio för att visa upp både personlighet, kompetens och tekniska färdigheter på ett tydligt och visuellt tilltalande sätt.

När jag påbörjade detta projekt var min erfarenhet inom html och CSS inte värst gedigen. Därför stötte jag på en hel del svårigheter och frågetecken på vägen. Men alla dessa svårigheter gjorde alltid att jag fick stanna upp och analysera min kod tills problemet var löst, och som tur känner jag att jag kommit ut på andra sidan. En av de svåraste momenten för mig skulle jag säga var “about me” sektionen där jag använde mig av ett antal “divs” för att få till “work experience” delen så att den såg ut som den skulle men samtidigt vara responsiv enligt Figma exemplet. Det som var mest klurigt var just hur man omvandlade “location” så att den flög ner från rad till column för mobilversionen samtidigt som "datum" var kvar som en row på höger sida. Det var verkligen ett moment där jag fick testa mig fram där huvudvärken började kicka in rejält. Men när jag äntligen hittade lösningen var det verkligen en givande känsla.

En styrka jag känner att jag har i mitt projekt är responsiviteten då den är från mina tester är responsiv i alla lägen på alla sidor utan problem. Något jag även är stålt över är de animationer och funktioner jag lyckats få till i form av “dark-mode” knapp och “hamburgar-meny” som visas och göms. Bristen på min webbsidan blir tyvärr att jag för att kunna skapa dessa funktioner, använt mig av javascript vilket underlättade det till skillnad från ifall jag skulle göra dem i enbart CSS. Vilket jag tror skulle vara möjligt men där jag inte kände mig lika kvalificerad för att ha tid och ork till att göra. Men som tur löste jag alla elementen från figma designen först i enbart CSS och HTML och påbörjade dessa "extra" funktioner i javascript först i slutet av projektet för att göra min sida lite roligare.

Vad är HTML och dess roll inom frontend? Vad menas med semantiskt html?

HTML står för HyperText Markup Language och är den strukturen för alla webbsidor och används för att skapa allt innehåll i form av t.ex. paragrafer, rubriker och bilder. Semantisk html innebär html som använder rätt element för rätt uppgifter och innehåller bra struktur. Det hjälper även webbläsaren att läsa in webbsidan korrekt vilket är viktigt inom SEO. Inom frontend fungerar HTML som skelettet i webbsidans uppbyggnad

Vad är CSS och dess roll inom frontend?

CSS står för Cascading Style Sheets och används för att styla och formatera HTML-element på en webbsida. Inom frontend bestämmer CSS utseendet och layouten, inklusive färger, typsnitt, avstånd, storlekar, positionering och andra visuella detaljer. Om HTML fungerar som sidans skelett, fungerar CSS som dess utseende och kläder vilket gör innehållet både attraktivt och lätt att läsa.
CSS gör det också möjligt att skapa responsiva och dynamiska webbsidor som anpassar sig till olika skärmstorlekar och enheter, ofta i kombination med tekniker som Flexbox, CSS Grid, media queries och relativa måttenheter. Genom CSS kan utvecklare skapa webbsidor som både är funktionella, visuellt tilltalande och användarvänliga överallt.
Vad menas med responsiv design samt med vilka metoder man kan utveckla responsiva sidor?

Responsiv design innebär att en webbsida automatiskt anpassar sitt utseende och innehåll efter olika skärmstorlekar och enheter såsom mobil, surfplatta och dator. Syftet är att sidan ska vara lika användarvänlig och funktionell oavsett enhet vilket förbättrar både användarupplevelsen och tillgängligheten.
För att skapa responsiva sidor används flera metoder inom webbutveckling. En vanlig strategi är mobile-first design, där man först designar och kodar för små skärmar och sedan anpassar layouten för större skärmar. Där används ofta CSS Media Queries för att definiera brytpunkter där layout, storlek och positioner ändras för att passa det nya formatet.
Andra viktiga tekniker för responsiv design är flexibla layouter med Flexbox eller CSS Grid, samt användning av relativa måttenheter som %, em och rem istället för fasta pixlar. Tillsammans gör dessa metoder det möjligt att skapa webbsidor som är smidiga, skalbara och visuellt konsekventa på alla typer av enheter.

Beskriv några konkreta tekniker enligt WCAG som kan förbättra en webbplats tillgänglighet

Genom att använda alternativtext (alt-text) på bilder kan skärmläsare beskriva bildinnehållet för personer med synnedsättning, vilket gör webbplatsen mer tillgänglig. En tydlig kontrast mellan text och bakgrund förbättrar läsbarheten för alla, särskilt för personer med nedsatt syn eller färgblindhet. Tangentbordsnavigering är också viktig, eftersom det gör att alla funktioner på sidan kan nås utan mus, vilket underlättar för användare med motoriska begränsningar.
Dessutom bidrar semantisk HTML, t.ex. <header>, <main>, <nav> och <button> tillsammans med beskrivande länkar och rubriker till att webbplatsens struktur blir lätt att förstå både för människor och hjälpmedel som skärmläsare. Genom att kombinera dessa tekniker skapas en mer inkluderande, användarvänlig och tillgänglig webbplats som fungerar för en bredare målgrupp.
Redogör för några av de viktigaste UX/UI-principerna inom frontend?

Några av de viktigaste UX/UI-principerna inom frontend handlar om att skapa tydliga, intuitiva och användarvänliga gränssnitt. En central princip är konsekvens vilket innebär att färger, ikoner och layouts används på ett enhetligt sätt för att göra sidan lätt att förstå. Visuell hierarki hjälper användaren att snabbt uppfatta vad som är viktigast genom tydliga kontraster, rubriknivåer och placering. Responsiv design säkerställer att sidan fungerar och ser bra ut på alla skärmstorlekar medan feedback t.ex. vid klick eller formulärinmatning visar att systemet reagerar på användarens handlingar. Slutligen är enkelhet och tillgänglighet avgörande för att alla användare, oavsett erfarenhet eller funktionsförmåga, ska kunna navigera och förstå innehållet utan hinder.
